How we built it

Figma MCP
레이아웃 설계 시 Figma MCP로 디자인 컨텍스트를 직접 참조하며 개발

컴포넌트 구조·색상·간격을 코드에 즉시 반영해 디자인-개발 간 오차를 최소화했어요
시안 비교 워크플로우
UI 요소마다 A안 / B안 시안을 나란히 생성 후 비교

담당자 컬럼·상태 뱃지·테이블 정렬 등 세부 UX를 반복 검토하며 최적안을 선택했어요
gs-os-ontology MCP
게임 목록 · 문서 링크를 온톨로지 MCP로 실시간 연동

GDD·Math Doc·Sound·연출 문서를 게임명 하나로 자동 매핑 — 수동 관리 불필요
Claude AI (Cowork) 와 페어 프로그래밍으로 전체 개발
기획 → 시안 → 코드 → 검증까지 AI와 협업 — 혼자서도 풀스택 QA 툴 완성
gs-os-ontology MCP 활용
게임 메타데이터를 코드 없이 연결하다
1
서버 시작 시 전체 게임 목록 자동 캐싱 search_games 툴로 스타트업 시 모든 게임 데이터를 메모리에 올려요 — 검색·드롭다운이 즉각 응답
2
게임명 → 문서 링크 원클릭 조회 get_game 툴로 GDD·Math Doc·Sound Sheet·연출 링크를 게임명 하나로 자동 반환 — TC 현황 카드 클릭 한 번으로 문서 패널 오픈
3
Drive 검색과 온톨로지 결과를 병합 · 우선순위 부여 온톨로지 결과(from_ontology: true)를 Drive 검색 결과보다 상위에 노출 — 정확도 높은 공식 링크가 항상 먼저 표시
4
통합 검색 키워드 확장에도 활용 게임 이름·코드·별칭을 온톨로지 기반으로 확장해 오탈자·약어 검색도 정확히 매칭

Goal
QA 팀 내부 공용 도구로 자리잡기
1차 목표 — QA 팀원 전체가 동일한 허브에서 업무
현재 — 개인 로컬 환경 localhost:8000 에서 운영 중 · QA 담당자 개인이 사용 · 팀 공유 불가
